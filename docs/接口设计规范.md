# 接口设计规范

## 1. URL 设计

### 1.1 基础路径
所有 API 以 `/api` 开头。单体项目/内部系统，暂不加入版本号。

### 1.2 RESTful 风格

| 操作 | HTTP 方法 | URL 示例 |
|------|-----------|----------|
| 查询列表 | GET | `/api/users?page=1&size=10` |
| 查询单个 | GET | `/api/users/1` |
| 新增 | POST | `/api/users` |
| 更新 | PUT | `/api/users/1` |
| 删除 | DELETE | `/api/users/1` |

### 1.3 资源命名
- 使用复数名词，小写字母 + 连字符（-）
- 示例：`/api/competition-achievements`, `/api/review-records`

## 2. 统一响应格式

所有接口返回以下 JSON 结构：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

### 2.1 状态码约定

| code | 含义 |
|------|------|
| 200 | 操作成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或 Token 过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 2.2 data 字段规范
- 单条数据：`data` 为对象 `{ ... }`
- 列表数据：`data` 为 `{ "list": [...], "total": 100, "page": 1, "size": 10 }`
- 无数据返回：`data` 为 `null`

### 2.3 分页参数
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| page | int | 1 | 页码，从 1 开始 |
| size | int | 10 | 每页条数，最大不超过 100 |

## 3. 后端 ApiResponse 工具类（骨架）

```java
package com.competition.dto;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

@Data
@AllArgsConstructor
@NoArgsConstructor
public class ApiResponse<T> {
    private int code;
    private String message;
    private T data;

    public static <T> ApiResponse<T> success(T data) {
        return new ApiResponse<>(200, "操作成功", data);
    }

    public static <T> ApiResponse<T> error(int code, String message) {
        return new ApiResponse<>(code, message, null);
    }
}
```

## 4. 前端 Axios 配置（骨架）

```js
// src/api/request.js
import axios from 'axios'
import router from '@/router'

const request = axios.create({
  baseURL: 'http://localhost:8080/api',
  timeout: 10000
})

request.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

request.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response?.status === 401) router.push('/login')
    return Promise.reject(error)
  }
)

export default request
```

## 5. 认证接口

### 5.1 用户注册

```
POST /api/auth/register
Content-Type: application/json
```

**请求参数：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | String | 是 | 用户名，3-50个字符 |
| password | String | 是 | 密码，BCrypt加密存储 |
| realName | String | 是 | 真实姓名 |
| email | String | 是 | 邮箱 |
| studentId | String | 否 | 学号/工号 |
| college | String | 否 | 所属学院 |

**响应示例：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": null
}
```

**错误响应：**
```json
{
  "code": 400,
  "message": "用户名已存在",
  "data": null
}
```

**说明：** 注册用户默认角色为 `STUDENT`。新注册账号默认为启用状态。

### 5.2 用户登录

```
POST /api/auth/login
Content-Type: application/json
```

**请求参数：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |

**响应示例：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "userInfo": {
      "id": 1,
      "username": "zhangsan",
      "realName": "张三",
      "role": "STUDENT"
    }
  }
}
```

**错误响应 (401)：**
```json
{
  "code": 401,
  "message": "用户名或密码错误",
  "data": null
}
```

### 5.3 获取当前用户信息

```
GET /api/auth/current
Authorization: Bearer {token}
```

**请求头：**
| 字段 | 值 | 说明 |
|------|-----|------|
| Authorization | Bearer {JWT token} | 登录时获取的访问令牌 |

**响应示例：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1,
    "username": "zhangsan",
    "realName": "张三",
    "role": "STUDENT"
  }
}
```

**说明：** Token 有效期 7 天。前端应在 Token 过期前重新登录。401 响应会清除前端本地存储的 Token。
