# 需求确认文档

## 1. 项目名称
竞赛科研管理系统

## 2. 目标用户与角色

| 角色 | 职责 |
|------|------|
| 学生 | 注册/登录、提交竞赛/科研成果、查看审核状态 |
| 教师 | 注册/登录、提交竞赛/科研成果、查看审核状态 |
| 科研秘书 | 审核学生/教师提交的成果、填写审核意见、提交给领导终审 |
| 学院领导 | 最终审核、归档成果、查看统计报表 |
| 系统管理员 | 用户管理（账号启用/禁用）、系统日志查看、数据维护 |

## 3. 核心功能模块

### 3.1 用户模块
- 注册：学号/工号 + 邮箱（限 @xxx.edu.cn 后缀） + 密码
- 登录：账号 + 密码，返回 JWT Token
- 个人信息修改：姓名、联系方式等

### 3.2 成果申报模块
- [x] 竞赛成果：竞赛名称、获奖等级、参赛时间、证明材料上传 ✅
- [x] 大创项目：项目名称、立项级别、立项类型、指导教师、立项人员、立项时间、证明材料上传 ✅
- [ ] 科研成果：论文标题、期刊/会议名称、发表时间、DOI、证明材料上传
- 草稿保存 + 正式提交（状态：待审核/领导审核中/已归档/已驳回）
- 文件上传：支持 PDF / 图片 / ZIP，大小限制 10MB

### 3.3 审核流程模块
- [x] 科研秘书一审：通过/退回修改/驳回，填写审核意见 ✅
- [x] 学院领导终审：通过/驳回，填写终审意见 ✅
- [x] 审核状态实时更新，系统消息通知 ✅
- [ ] 邮件通知（后期扩展）

### 3.4 成果展示模块
- 按年份、类型、等级筛选
- 个人成果列表（自己提交的）
- 全院成果统计面板（管理员/领导可见）

### 3.5 系统管理模块
- 用户角色分配
- 操作日志审计
- 数据导出（Excel）

## 4. 非功能需求
- 响应式布局（PC 端优先）
- 接口统一返回格式 {code, message, data}
- 文件上传支持 PDF / 图片（大小限制 10MB）
- 密码加密存储（BCrypt）
