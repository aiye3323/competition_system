# 数据库设计规范

## 1. 命名规范

### 1.1 表命名
- 格式：`t_` + 下划线分隔的小写单词
- 示例：`t_user`, `t_competition_achievement`, `t_review_record`

### 1.2 字段命名
- 格式：`f_` + 下划线分隔的小写单词
- 示例：`f_id`, `f_user_name`, `f_create_time`
- 布尔字段：`f_is_deleted`, `f_is_approved`

### 1.3 索引命名
- 唯一索引：`uk_` + 字段简称（如 `uk_user_email`）
- 普通索引：`idx_` + 字段简称（如 `idx_submit_time`）

## 2. 必备基础字段（每张表必须包含）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| f_id | BIGINT | 主键，AUTO_INCREMENT |
| f_create_time | DATETIME | 创建时间，DEFAULT CURRENT_TIMESTAMP |
| f_update_time | DATETIME | 更新时间，ON UPDATE CURRENT_TIMESTAMP |
| f_is_deleted | TINYINT(1) | 逻辑删除，0=正常 1=已删除，DEFAULT 0 |

## 3. 字段类型约定

| 数据类别 | MySQL 类型 | 说明 |
|----------|------------|------|
| 主键/外键 | BIGINT | 自增主键，应用层关联外键 |
| 短文本（<=255字符） | VARCHAR(N) | 指定合适长度 |
| 长文本（>255字符） | TEXT | 审核意见、成果描述等 |
| 日期时间 | DATETIME | 统一使用 DATETIME，不用 TIMESTAMP |
| 金额/小数 | DECIMAL(10,2) | 精确小数 |
| 布尔标记 | TINYINT(1) | 0/1 表示 |
| 枚举状态 | VARCHAR(20) | 用字符串，不用数据库 ENUM 类型 |

## 4. 外键策略
- 应用层（JPA）维护实体关联关系
- **不建数据库级外键约束**（方便分库分表扩展）
- 外键字段命名：`f_` + 关联表名 + `_id`，如 `f_user_id` 关联 `t_user.f_id`

## 5. 字符集与排序规则
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_unicode_ci`
- 引擎：InnoDB

## 6. 建表示例

```sql
CREATE TABLE `t_user` (
    `f_id` BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键',
    `f_user_name` VARCHAR(50) NOT NULL COMMENT '用户名',
    `f_password` VARCHAR(255) NOT NULL COMMENT '密码（BCrypt加密）',
    `f_email` VARCHAR(100) NOT NULL COMMENT '邮箱',
    `f_role` VARCHAR(20) NOT NULL DEFAULT 'STUDENT'
        COMMENT '角色：STUDENT/TEACHER/SECRETARY/LEADER/ADMIN',
    `f_status` TINYINT(1) NOT NULL DEFAULT 1 COMMENT '0=禁用 1=启用',
    `f_create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `f_update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `f_is_deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '逻辑删除',
    UNIQUE KEY `uk_user_email` (`f_email`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

## 建表SQL

### 1. t_user（用户表）

```sql
CREATE TABLE `t_user` (
    `f_id`              BIGINT          AUTO_INCREMENT PRIMARY KEY
        COMMENT '主键ID',
    `f_username`        VARCHAR(50)     NOT NULL
        COMMENT '用户名，登录账号',
    `f_password`        VARCHAR(255)    NOT NULL
        COMMENT '密码，BCrypt加密存储',
    `f_real_name`       VARCHAR(50)     DEFAULT NULL
        COMMENT '真实姓名',
    `f_role`            VARCHAR(20)     NOT NULL DEFAULT 'STUDENT'
        COMMENT '角色：STUDENT/TEACHER/SECRETARY/LEADER/ADMIN',
    `f_email`           VARCHAR(100)    NOT NULL
        COMMENT '邮箱，可用于登录和通知',
    `f_student_id`      VARCHAR(50)     DEFAULT NULL
        COMMENT '学号（学生）/工号（教师/秘书/领导）',
    `f_college`         VARCHAR(100)    DEFAULT NULL
        COMMENT '所属学院',
    `f_register_time`   DATETIME        DEFAULT NULL
        COMMENT '注册时间',
    `f_last_login_time` DATETIME        DEFAULT NULL
        COMMENT '最后登录时间',
    `f_status`          TINYINT(1)      NOT NULL DEFAULT 1
        COMMENT '账号状态：0=禁用 1=正常',
    `f_create_time`     DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP
        COMMENT '记录创建时间',
    `f_update_time`     DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
        COMMENT '记录更新时间',
    `f_is_deleted`      TINYINT(1)      NOT NULL DEFAULT 0
        COMMENT '逻辑删除：0=正常 1=已删除',

    UNIQUE KEY  `uk_username`     (`f_username`),
    UNIQUE KEY  `uk_email`        (`f_email`),
    INDEX       `idx_role`        (`f_role`),
    INDEX       `idx_status`      (`f_status`),
    INDEX       `idx_student_id`  (`f_student_id`),
    INDEX       `idx_college`     (`f_college`)

) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```
