# 科研竞赛管理系统 — 文件命名规范

## 一、命名格式总览

```
{成果类型}-{项目名称}-{申报人}-{材料类型}.{扩展名}
```

| 组件 | 说明 |
|------|------|
| 成果类型 | 学科竞赛 / 创新项目 / 学术论文 / 软件著作 |
| 项目名称 | 竞赛名称、项目名称、期刊名称、软件名称 |
| 申报人 | 取第一人，多人加"等" |
| 材料类型 | 获奖证书 / 立项申报书 / 结题材料 等 |
| 扩展名 | .pdf / .png / .docx / .zip 等 |

---

## 二、各类型命名规则

### 2.1 学科竞赛

```
学科竞赛-{竞赛名称}-{参赛选手}[-{材料类型}].扩展名
```

| 材料类型 | 示例 |
|---------|------|
| 获奖证书 | 学科竞赛-全国大学生数学竞赛-张三-获奖证书.png |
| 现场合影 | 学科竞赛-挑战杯-李四等-现场合影.jpg |
| 其他材料 | 学科竞赛-互联网+大赛-王五-其他材料.zip |

### 2.2 大学生创新训练计划项目

```
创新项目-{项目名称}-{立项人员}[-{材料类型}].扩展名
```

| 材料类型 | 示例 |
|---------|------|
| 立项申报书 | 创新项目-智能机器人研发-李四-立项申报书.pdf |
| 结题材料 | 创新项目-智慧农业系统-赵六等-结题材料.zip |
| 结题证书 | 创新项目-AI语音助手-钱七-结题证书.png |

### 2.3 学术论文

```
学术论文-{期刊名称}-{作者列表}[-{材料类型}].扩展名
```

| 材料类型 | 示例 |
|---------|------|
| 投稿初稿 | 学术论文-Knowledge-Based Systems-王五等-投稿初稿.pdf |
| 录用终稿 | 学术论文-计算机学报-张三-录用终稿.pdf |
| 审稿意见 | 学术论文-IEEE TNNLS-李四等-审稿意见.pdf |

### 2.4 软件著作

```
软件著作-{软件名称}-{著作权人}[-{材料类型}].扩展名
```

| 材料类型 | 示例 |
|---------|------|
| 申报材料 | 软件著作-图书管理系统-赵六-申报材料.zip |
| 证书扫描件 | 软件著作-在线考试平台-钱七-证书扫描件.png |

## 三、特殊规则

| 规则 | 处理方式 |
|------|---------|
| 多人参与 | `getFirstPerson()` — 逗号/分号/顿号分隔，取第一位 + "等" |
| 文件名过长 | `truncate(text, 30)` — 超过 30 字符截断 |
| 非法字符 | `sanitize()` — `\ / : * ? " < > |` → `_` |
| 重名冲突 | `dedupFileName(name, index)` — 加 `(1)` `(2)` 后缀 |
| 中文编码 | 下载时 `encodeURIComponent(fileName)` |

## 四、代码位置

| 文件 | 说明 |
|------|------|
| `frontend/src/utils/fileNaming.js` | 核心命名工具函数 |

### 主要 API

```js
import {
  generateFileName,           // 生成规范文件名
  generateBatchZipName,       // 批量 ZIP 名
  generateAllZipName,         // 全部材料 ZIP 名
  getFirstPerson,             // 取第一人 + "等"
  inferMaterialType,          // 从原始文件名推断材料类型
  sanitize,                   // 清理非法字符
  truncate,                   // 截断
  dedupFileName,              // 重名去重
  assignDownloadNames,        // 批量分配不重名
  downloadBlob                // blob 下载
} from '@/utils/fileNaming'
```
