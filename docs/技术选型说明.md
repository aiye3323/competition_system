# 技术选型说明

## 1. 总体架构
B/S 架构，前后端分离。前端 SPA 应用，后端 RESTful API 服务。

## 2. 前端技术栈

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| Vue | 3.x | 生态成熟，中文文档完善，国内企业广泛采用 |
| Vite | 5.x | 比 Webpack 快 10 倍以上的开发服务器启动速度 |
| Element Plus | 2.x | Vue 3 官方推荐的桌面端 UI 组件库，适合管理后台 |
| Axios | 1.x | Promise 风格 HTTP 客户端，支持请求/响应拦截器 |
| Vue Router | 4.x | Vue 3 官方路由方案，支持路由守卫 |

## 3. 后端技术栈

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| Spring Boot | 3.2.5 | 企业级 Java 框架，生态完整，与 JDK 17 兼容 |
| Spring Data JPA | - | ORM 框架，简化 CRUD，自动建表，减少 SQL 编写 |
| Spring Security | 6.x | 认证授权框架，支持 JWT + 角色权限控制 |
| MySQL | 8.0 | 开源关系型数据库，utf8mb4 完整支持中文 |
| Lombok | 1.18.x | 减少样板代码（Getter/Setter/Constructor） |
| Maven | 3.x | Java 项目构建和依赖管理 |

## 4. 辅助方案

| 场景 | 方案 | 说明 |
|------|------|------|
| 文件上传 | Spring MultipartFile + 本地存储 | 证明材料上传，后期可扩展 OSS |
| 图表统计 | ECharts 5.x | 成果统计面板展示，第 6 阶段引入 |
| Excel 导出 | Apache POI | 数据导出，第 6 阶段引入 |
| API 测试 | Postman / IDEA HTTP Client | 接口调试 |
| 代码版本 | Git（本地仓库） | 暂不推送到远程 |

## 5. 不选方案及理由

| 不选 | 理由 |
|------|------|
| Oracle / SQL Server | 商业授权，学习环境不适用 |
| MongoDB | 本项目数据关系强（用户-成果-审核），更适合关系型 |
| React / Angular | Vue 学习曲线更低，文档中文支持更好 |
| MyBatis | JPA 更适合快速原型开发，后期可按需引入 MyBatis |
| Element UI（Vue 2） | 本项目使用 Vue 3，不兼容 |
