### 9. 修复文件预览和下载报错
- 报错原因：`Paths.get(uploadPath, storagePath)` 每次都重新拼接，中文路径下拼接结果不稳定
- 修复方案：改为启动时用 `Paths.get(uploadPath).toAbsolutePath().normalize()` 缓存 `baseUploadPath`，后续用 `baseUploadPath.resolve(storagePath)`
- FileController 增加 try-catch：文件不存在返回 400 JSON 错误，记录不存在返回 404
- 增加路径遍历安全检查：`filePath.startsWith(baseUploadPath)`
- 测试结果：上传 → 预览 HTTP 200 ✅，下载 HTTP 200 ✅，不存在文件返回 404 JSON ✅
